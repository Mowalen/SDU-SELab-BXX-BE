### 2. CASE（Computer-Aided Software Engineering, 计算机辅助软件工程）工具调研及应用

&ensp;&ensp;小组分工搜索各种主流软件工程技术网站，调研有哪些流行的CASE工具，分析它们的用途、技术特点。
#### 1. case工具概念
    ``` 
    case工具原来指用来支持管理信息系统开发的、由各种计算机辅助软件和工具组成的大型综合性软件开发环境。
    随着各种工具和软件技术的产生、发展、完善和不断集成，逐步由单纯的辅助开发工具环境转化为一种相对独立的方法论
    ```
#### 2. CASE工具举例：

1. **需求分析工具**
- **IBM Rational RequisitePro**
   - **用途**：用于收集、组织、追踪和管理软件项目的需求。
   - **特点**：提供强大的需求管理和追溯能力，支持需求变更管理、需求版本控制、需求与模型关联等。
   - **优点**：结构化管理需求，提高需求可追踪性，有助于确保需求一致性。
   - **缺点**：可能对小型项目来说过于复杂，学习曲线较陡峭。
2. **UML建模工具**
- **Rational Rose**
   - **用途**：创建和编辑统一建模语言(UML)图表，进行系统分析与设计。
   - **特点**：支持多种UML视图，包括类图、序列图、状态图等，可用于面向对象的设计。
   - **优点**：集成度高，能够从高层次设计细化到代码生成。
   - **缺点**：随着其他更轻量级工具的发展，Rose的市场占有率有所下降，且对于非OO项目适应性有限。
 
- **Visual Paradigm**：
  - **用途**：Visual Paradigm 是一个跨平台的 UML 建模工具，支持各种 UML 图表和建模技术。
  - **特点**：提供了丰富的建模工具和模板，支持需求管理、系统分析、设计和实现等不同阶段的建模需求。
  - **优点**：具有直观的用户界面和易于使用的功能，可帮助团队协作和沟通，提高开发效率。
  - **缺点**：部分高级功能可能需要购买专业版或高级许可证。

3. **数据库建模工具**
- **PowerDesigner**
   - **用途**：设计和实现数据库结构，包括概念数据模型、逻辑数据模型和物理数据模型设计。
   - **特点**：支持多种数据库平台，具备强大的数据模型转换和逆向工程功能。
   - **优点**：全面的数据建模支持，方便数据库管理员和开发人员协同工作。
   - **缺点**：高级功能较为复杂，对于简单项目可能显得过于强大，且成本相对较高。

4. **版本控制/配置管理工具**
- **Git**
   - **用途**：管理源代码历史版本，支持团队协作，解决代码合并冲突问题。
   - **特点**：分布式版本控制系统，速度快，分支管理灵活。
   - **优点**：开源免费，社区活跃，广泛应用于各类项目。
   - **缺点**：对于初学者有一定的学习门槛，大型项目管理需要良好的流程规划。

- **SVN (Subversion)**
   - **用途**：集中式版本控制系统，用于存储和跟踪文件及目录的修改历史。
   - **特点**：服务器集中管理，适合传统开发模式，易于理解和操作。
   - **优点**：稳定可靠，普及率高。
   - **缺点**：相较于Git，分支和合并操作不够灵活。

5. **项目管理和协同工作工具**
- **Jira**
   - **用途**：管理软件开发过程中的任务分配、bug跟踪、敏捷开发迭代等。
   - **特点**：高度可定制，适用于敏捷和瀑布等多种开发方法。
   - **优点**：强大的工单系统，集成了很多第三方工具和服务。
   - **缺点**：对于小团队或简单项目，可能存在功能过剩的问题。

6. **集成开发环境（IDE）**
- **Eclipse / IntelliJ IDEA**
   - **用途**：编写、调试、构建和部署软件应用程序。
   - **特点**：提供丰富的插件支持，集成代码编辑、编译、调试等功能。
   - **优点**：极大提高开发效率，支持多语言和框架。
   - **缺点**：资源消耗较大，对于低端设备运行可能稍显吃力。
 
- **Visual Studio**
  - **用途**：主要用于开发Microsoft平台的软件应用程序，包括Windows桌面应用、Web应用、移动应用等。
  - **特点**：提供了丰富的工具和功能，包括代码编辑器、调试器、图形化界面设计器等。
  - **优点**：紧密集成了各种开发工具和服务，提高了开发效率和便捷性。
  - **缺点**：主要面向Microsoft生态系统，对于其他平台的开发支持可能相对较弱。

- **PyCharm**
  - **用途**：主要用于Python开发，支持各种Python项目的开发、调试和部署。
  - **特点**：提供了智能代码编辑、语法高亮、代码导航、代码重构等功能，集成了调试器、版本控制和虚拟环境管理工具。
  - **优点**：针对Python开发进行了优化，提供了丰富的功能和插件支持，可大大提高开发效率。
  - **缺点**：对于初学者来说，可能需要一定时间来熟悉其功能和操作方式。

7. **测试工具**
- **Selenium**
   - **用途**：自动化Web应用的功能测试。
   - **特点**：跨浏览器支持，可编写脚本模拟用户交互行为。
   - **优点**：开源，支持多种编程语言编写测试脚本。
   - **缺点**：需要编写测试脚本，对自动化测试技能有一定要求。

 8. **静态代码分析工具**
- **SonarQube**
  - **用途**：进行静态代码分析，检测代码质量和安全漏洞。
  - **特点**：支持多种编程语言，提供可视化报告和实时反馈。
  - **优点**：帮助团队发现和修复代码中的问题，提升代码质量和可维护性。
  - **缺点**：配置和管理较为复杂，需要一定的学习和实践经验。

9. **持续集成工具**
- **Jenkins**
  - **用途**：自动化构建、测试和部署软件项目。
  - **特点**：可扩展性强，支持各种插件和集成。
  - **优点**：提高开发团队的效率，减少手动操作，保证软件质量。
  - **缺点**：需要一定的配置和维护成本，初学者上手可能有一定难度。

10. **容器编排工具**
- Kubernetes
  - **用途**：管理容器化应用程序的部署、扩展和操作。
  - **特点**：高度可扩展，支持自动化容器部署和资源管理。
  - **优点**：提供强大的容器编排能力，适用于大规模和复杂的分布式系统。
  - **缺点**：学习曲线较陡，需要理解容器化和分布式系统的基本概念。


### 3. 选择适合的CASE工具，补充、完善所做的可行性分析报告
&ensp;&ensp;小组根据调研情况讨论，结合自己的项目和拟使用的技术路线，选择了git作为版本控制工具，VsCode、Pycharm等软件作为集成开发环境,使用Visual Paradigm作为UML建模工具。

#####理由如下：
 
 **Git版本控制工具：**
    - 流行和广泛应用: Git是当今最流行的版本控制系统之一，被广泛应用于各种规模和类型的项目中，拥有庞大的用户群和活跃的社区。
    - 分布式版本控制: Git是一种分布式版本控制系统，每个开发者都拥有完整的版本库，可以在本地进行版本控制操作，这大大提高了开发效率和灵活性。
    - 强大的分支管理: Git的分支管理功能非常强大，开发者可以轻松地创建、合并、切换分支，有助于并行开发和团队协作。
    - 开源免费: Git是开源软件，免费使用，而且有许多优秀的图形化界面和第三方工具可供选择，便于使用和集成。
   
**VSCode和PyCharm集成开发环境：**
   - 专注于语言: VSCode和PyCharm分别专注于多种语言和Python语言的开发。VSCode具有对多种编程语言的支持，而PyCharm专注于Python开发，提供了针对Python的丰富功能和优化。
   - 丰富的插件生态系统: 两者都有强大的插件生态系统，可以根据需要轻松扩展功能，满足不同开发需求。
   - 智能代码补全和语法检查: 提供智能代码补全、语法高亮、代码导航等功能，有助于提高编码效率和代码质量。
   - 集成版本控制: VSCode和PyCharm都集成了对Git的支持，可以直接在IDE中进行版本控制操作，简化了开发流程。
   - 强大的调试功能: 提供了强大的调试器，可以方便地进行代码调试和错误排查，有助于提高代码质量和开发效率。
   - 友好的用户界面: 两者都具有直观友好的用户界面，易于上手，适合不同水平的开发者使用。

**Visual ParadigmUML建模工具：**
  - 丰富的功能和工具：Visual Paradigm 提供了丰富的 UML 建模工具和功能，涵盖了多种 UML 图表和建模技术，包括类图、用例图、时序图等。它还支持需求管理、系统分析、设计和实现等不同阶段的建模需求。
  - 直观的用户界面：Visual Paradigm 具有直观的用户界面，易于使用和操作。它提供了易于理解的工具栏、菜单和操作界面，使得用户可以快速上手，并且可以轻松地创建、编辑和管理 UML 图表。
  - 团队协作和沟通：Visual Paradigm 支持团队协作和实时共享，团队成员可以通过该工具共同编辑和查看 UML 图表，实现沟通和合作。这有助于团队成员之间的交流和协作，提高了团队的工作效率和质量。
  - 跨平台使用：Visual Paradigm 可以在多个操作系统上运行，包括 Windows、Mac 和 Linux，这使得团队成员可以在不同的平台上使用同一个工具进行建模和开发，增强了灵活性和可访问性。
  - 持续更新和改进：Visual Paradigm 持续更新和改进其产品，提供了稳定的版本和新功能的迭代，使得用户始终可以使用最新的技术和工具来进行 UML 建模和开发。
