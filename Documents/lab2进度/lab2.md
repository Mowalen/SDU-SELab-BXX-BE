### 2. CASE（Computer-Aided Software Engineering, 计算机辅助软件工程）工具调研及应用

&ensp;&ensp;小组分工搜索各种主流软件工程技术网站，调研有哪些流行的CASE工具，分析它们的用途、技术特点。
#### 1. case工具概念
    ``` 
    case工具原来指用来支持管理信息系统开发的、由各种计算机辅助软件和工具组成的大型综合性软件开发环境。
    随着各种工具和软件技术的产生、发展、完善和不断集成，逐步由单纯的辅助开发工具环境转化为一种相对独立的方法论
    ```
#### 2. CASE工具举例：

1. **需求分析工具**
- **IBM Rational RequisitePro**
   - **用途**：用于收集、组织、追踪和管理软件项目的需求。
   - **特点**：提供强大的需求管理和追溯能力，支持需求变更管理、需求版本控制、需求与模型关联等。
   - **优点**：结构化管理需求，提高需求可追踪性，有助于确保需求一致性。
   - **缺点**：可能对小型项目来说过于复杂，学习曲线较陡峭。
2. **UML建模工具**
- **Rational Rose**
   - **用途**：创建和编辑统一建模语言(UML)图表，进行系统分析与设计。
   - **特点**：支持多种UML视图，包括类图、序列图、状态图等，可用于面向对象的设计。
   - **优点**：集成度高，能够从高层次设计细化到代码生成。
   - **缺点**：随着其他更轻量级工具的发展，Rose的市场占有率有所下降，且对于非OO项目适应性有限。

3. **数据库建模工具**
- **PowerDesigner**
   - **用途**：设计和实现数据库结构，包括概念数据模型、逻辑数据模型和物理数据模型设计。
   - **特点**：支持多种数据库平台，具备强大的数据模型转换和逆向工程功能。
   - **优点**：全面的数据建模支持，方便数据库管理员和开发人员协同工作。
   - **缺点**：高级功能较为复杂，对于简单项目可能显得过于强大，且成本相对较高。

4. **版本控制/配置管理工具**
- **Git**
   - **用途**：管理源代码历史版本，支持团队协作，解决代码合并冲突问题。
   - **特点**：分布式版本控制系统，速度快，分支管理灵活。
   - **优点**：开源免费，社区活跃，广泛应用于各类项目。
   - **缺点**：对于初学者有一定的学习门槛，大型项目管理需要良好的流程规划。

- **SVN (Subversion)**
   - **用途**：集中式版本控制系统，用于存储和跟踪文件及目录的修改历史。
   - **特点**：服务器集中管理，适合传统开发模式，易于理解和操作。
   - **优点**：稳定可靠，普及率高。
   - **缺点**：相较于Git，分支和合并操作不够灵活。

5. **项目管理和协同工作工具**
- **Jira**
   - **用途**：管理软件开发过程中的任务分配、bug跟踪、敏捷开发迭代等。
   - **特点**：高度可定制，适用于敏捷和瀑布等多种开发方法。
   - **优点**：强大的工单系统，集成了很多第三方工具和服务。
   - **缺点**：对于小团队或简单项目，可能存在功能过剩的问题。

6. **集成开发环境（IDE）**
- **Eclipse / IntelliJ IDEA**
   - **用途**：编写、调试、构建和部署软件应用程序。
   - **特点**：提供丰富的插件支持，集成代码编辑、编译、调试等功能。
   - **优点**：极大提高开发效率，支持多语言和框架。
   - **缺点**：资源消耗较大，对于低端设备运行可能稍显吃力。

7. **测试工具**
- **Selenium**
   - **用途**：自动化Web应用的功能测试。
   - **特点**：跨浏览器支持，可编写脚本模拟用户交互行为。
   - **优点**：开源，支持多种编程语言编写测试脚本。
   - **缺点**：需要编写测试脚本，对自动化测试技能有一定要求。
